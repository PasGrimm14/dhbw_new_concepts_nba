// 01 — Definition

Was ist
Cloud Computing?

Cloud Computing bezeichnet die Bereitstellung von IT-Ressourcen über das Internet – auf Abruf, nach Bedarf und als bezahlter Service.

🌐 NIST Definition

On-Demand Self-Service

Ressourcen wie Rechenleistung oder Speicher können jederzeit ohne menschliche Interaktion mit dem Anbieter bereitgestellt werden.

🔗 Zugang

Breiter Netzwerkzugang

Dienste sind über Standardmechanismen verfügbar und zugänglich von verschiedenen Geräten wie Laptops, Tablets oder Smartphones.

📦 Ressourcen

Ressourcen-Pooling

Die Ressourcen des Anbieters werden in einem Pool zusammengefasst und mehreren Kunden gleichzeitig dynamisch zugewiesen.

⚡ Elastizität

Schnelle Elastizität

Kapazitäten können elastisch bereitgestellt und freigegeben werden – oft automatisch, um der Nachfrage zu folgen.

// 02 — Vorteile

Warum die
Cloud?

Unternehmen weltweit migrieren in die Cloud – nicht ohne Grund.

💰

Kosteneffizienz

Keine teuren Vorabinvestitionen in Hardware. Pay-as-you-go: Zahlen nur für tatsächlich genutzte Ressourcen.

📈

Skalierbarkeit

Binnen Minuten von 10 auf 10.000 Server skalieren. Kapazität wächst mit dem Unternehmen – ohne Vorlaufzeit.

🛡️

Sicherheit & Compliance

Hyperscaler investieren Milliarden in Sicherheit. ISO 27001, SOC 2, DSGVO-Compliance out-of-the-box.

🌍

Globale Reichweite

Daten und Anwendungen in Rechenzentren weltweit verteilen. Niedrige Latenz für Nutzer rund um den Globus.

🔄

Agilität

Neue Umgebungen in Sekunden aufsetzen. Development, Test und Production parallel betreiben.

♻️

Wartungsfreiheit

Updates, Patches und Infrastruktur-Wartung übernimmt der Anbieter. Teams fokussieren auf Innovation.

// 07 — Anwendungsfälle

Databricks
Use Cases

Wie führende Unternehmen Databricks einsetzen.

USE CASE · 01

🏦 Finanzanalyse

Echtzeit-Risikoberechnungen auf Millionen von Transaktionen. Ein Großbank verarbeitet täglich 2 TB Transaktionsdaten mit Spark-Clustern, die on-demand hochskalieren.

USE CASE · 02

🛒 E-Commerce Empfehlungen

Personalisierungsmodelle trainieren auf dem gesamten Kaufverlauf. Cluster starten nachts für Batch-Training und fahren tagsüber für Inferenz auf wenige Nodes herunter.

USE CASE · 03

🏥 Healthcare Analytics

Anonymisierte Patientendaten in Delta Lake. Compliance-Teams können via Unity Catalog genau sehen, wer wann auf welche Daten zugegriffen hat.

USE CASE · 04

🚗 IoT & Streaming

Fahrzeugtelemetrie von 500.000 Autos in Echtzeit verarbeiten. Structured Streaming in Databricks empfängt, transformiert und speichert Daten mit sub-Sekunden-Latenz.

USE CASE · 05

🤖 Generative KI

Fine-Tuning von LLMs auf unternehmenseigenen Daten. DBRX, Databricks' eigenes Large Language Model, läuft vollständig in der eigenen Cloud-Infrastruktur.

USE CASE · 06

📊 Data Warehouse Migration

Unternehmen migrieren teure On-Premise-Warehouses zu Databricks SQL. 10x günstigere Kosten bei gleichzeitig höherer Performance durch Cloud-Elastizität.

On-Premise
vs. Cloud

Am Beispiel einer Databricks-ähnlichen Datenplattform – traditionell vs. cloud-nativ.

Kriterium On-Premise Cloud (Databricks)
Inbetriebnahme 6–18 Monate (Hardware beschaffen, aufbauen, konfigurieren) Workspace in <10 Minuten erstellt, sofort produktiv
Skalierung Monate Neue Server kaufen, liefern, einbauen Sekunden Auto-Scaling per API oder automatisch
Kosten Hohe CapEx (Hardware), fixe OpEx – egal ob genutzt oder nicht Reine OpEx, Pay-per-Use – Cluster stoppen = 0 Kosten
Updates & Patches Manuell Downtimes einplanen, eigenes Ops-Team Automatisch Databricks updated Platform im Hintergrund
Disaster Recovery Zweites Rechenzentrum nötig – doppelte Kosten Multi-Region-Replikation, automatische Failover
Globaler Zugang Eingeschränkt VPN, Latenz, Komplexität Nativ Zugang weltweit, SSO, Web-Browser
